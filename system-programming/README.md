# Системне програмування

Ця директорія містить мої лабораторні роботи та базову інфраструктуру для їх компіляції й запуску. Проєкт орієнтований на Linux-середовище, але його можна запускати і на macOS/Windows через Docker, якщо це потрібно для виконання умов завдання.

## Передумови

Перед початком роботи варто переконатися, що встановлено:

- **C++ компілятор** (рекомендовано: `g++` або `clang++`)
- **Make**
- (Опційно) **Docker**, якщо середовище не Linux, але потрібно запускати роботи саме в Linux-оточенні

На macOS можна встановити все через Homebrew:
```bash
brew install make gcc
```

## Компіляція та запуск

Щоб зібрати всі лабораторні:

```bash
make
```

Щоб зібрати окрему лабораторну, наприклад `lab1`:

```bash
make lab1
```

Усі скомпільовані бінарні файли зберігаються в директорії `build/`.

Щоб видалити згенеровані файли:

```bash
make clean
```

Після компіляції кожен бінарник можна запускати напряму, наприклад:

```bash
./build/lab1
```

## Запуск на Ubuntu з Docker (Linux оточення)

Docker використовується лише для ситуацій, коли оточення розробника не Linux, але лабораторні **мають бути виконані саме в Linux**. Для перевірки роботи це не є обов’язковим.

### Створення Docker-образу

```bash
docker build -t cpp_labs_env .
```

### Запуск контейнера

```bash
docker run -it --rm -v $(pwd):/work cpp_labs_env
```

У контейнері можна запускати ті ж команди `make`, що і локально.

## Структура проєкту

```
.
├── lab1
│   └── lab1_yaroshchuk_pi_22_1.cpp
├── Makefile
└── Dockerfile              # Опціональний спосіб запуску під Linux (Ubuntu)
```

## Коротко

* Потрібно лише Make + компілятор.
* Docker використовується як частина завдання, а не як обов’язкова інфраструктура.
* Усі результати збірки лежать у `build/`.
